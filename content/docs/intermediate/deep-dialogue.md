{{% alert icon="" context="info" %}}
ここでの「壁打ち」とは、自分の考えをAIにぶつけて、整理・深掘り・別角度からの視点を引き出すことを指します。
AIに正解を求めるのではなく、<strong>自分の思考を外に取り出して検討するプロセス</strong>です。

## 🎯 壁打ちの3つのパターン
---

大学業務での壁打ちは、大きく3つのパターンに分けられます。

| パターン | 目的 | 使いどころ |
|:---|:---|:---|
| **アイデアの構造化** | モヤモヤした考えを形にする | 企画の初期段階、方針検討 |
| **論点の洗い出し** | 見落としている観点を発見する | 政策案の検討、リスク分析 |
| **たたき台の改善** | 作成した案をブラッシュアップする | 企画書、研修計画、報告書の推敲 |

<br>

## 💡 パターン1：アイデアを構造化する
---

「やりたいことはあるけれど、まだ形になっていない」という段階で、AIに壁打ちしてみましょう。

### - 企画の種を形にする

```
相談があります。
学内で教職員向けの生成AI活用研修を企画したいのですが、
まだ漠然としたアイデアの段階です。

分かっていること：
- 対象は事務職員（約50名）
- 予算はほとんどない
- 年度内に1回は実施したい
- 参加者のITリテラシーにはかなりばらつきがある

この条件で、どんな研修が考えられるか、一緒に考えてもらえますか？
```
{{% alert icon="🤖" context="success" %}}
いくつかの方向性を考えてみました。条件を整理しながら進めましょう。

**方向性A：体験型ワークショップ（2時間 x 1回）**
- 実際に生成AIを触りながら、自分の業務で使えるプロンプトを作成する
- メリット：参加者が「使えそう」と実感しやすい
- 課題：2時間だと深い内容は難しい

**方向性B：段階別の少人数セッション（1時間 x 2回）**
- 初回：基礎編（生成AIとは何か、基本操作）
- 2回目：実践編（業務別の活用事例）
- メリット：リテラシーのばらつきに対応しやすい
- 課題：2回の日程調整が必要

**方向性C：自習型 + 相談会**
- 動画教材やマニュアルを配布し、自分のペースで学習
- 月1回の「相談会」で疑問を解消
- メリット：予算を最小限に抑えられる
- 課題：自主的な学習には個人差が出やすい

**確認したいこと：**
- 参加者が最も困っている業務は何ですか？
- 研修後のゴールイメージ（「全員が1つのプロンプトを使えるようになる」等）はありますか？
{{% /alert %}}

ここで重要なのは、AIの回答をそのまま採用することではありません。
**「確認したいこと」として返ってくる質問に答えていくことで、自分自身の考えも整理されていく**のがポイントです。

対話を続けてみましょう。

```
ありがとうございます。方向性Bが良さそうです。
参加者が困っていることとしては、メール文面の作成と会議資料の要約が多いと思います。
ゴールは「日常業務で1つ以上のタスクにAIを使えるようになる」くらいの感じです。

あと気になっているのは、「AIに仕事を奪われるのでは」という不安を持っている人が
結構いるので、その心理的なハードルをどう下げるかも考えたいです。
```

{{% alert icon="🤖" context="success" %}}
方向性Bをベースに、もう少し具体的にしてみます。

**研修プラン案：「はじめての生成AI活用」（全2回）**

**第1回：知る・触れる（1時間）**
1. 導入（10分）：AIは「便利な道具」であり、仕事を奪うものではないという位置づけの共有
   - 「電卓が登場しても経理の仕事はなくならなかった」等の身近な例で説明
2. 体験（30分）：全員でメール文面の下書きを生成AIに作ってもらう
3. 振り返り（20分）：使ってみた感想の共有、疑問点の解消

**第2回：使う・活かす（1時間）**
1. 前回の復習と質問対応（10分）
2. 実践（35分）：自分の実際の業務資料を使って要約を体験
3. まとめ（15分）：日常業務での活用アイデアを各自1つ発表

**心理的ハードルへの対応：**
- 研修の冒頭で「AIは下書きを作る道具。最終判断は必ず人間が行う」と明示する
- 「失敗しても大丈夫」な環境を作る（練習用のアカウントを用意する等）
- 参加者同士で体験を共有する時間を設け、「自分だけではない」安心感を作る

この方向で良さそうですか？調整したい部分があれば教えてください。
{{% /alert %}}

2回の対話で、漠然としたアイデアがかなり具体的な計画に近づきました。
もちろんここからさらに詳細を詰めていくことができます。

## 🔍 パターン2：論点を洗い出す
---

ある程度方針が固まった段階で、「見落としている観点はないか」をAIに確認してもらうパターンです。

### - 政策案の論点整理

```
以下の方針案について、賛成意見と反対意見の両方の立場から
論点を洗い出してください。
私が見落としている観点があれば指摘してほしいです。

方針案：
来年度から全学的にAIツールの利用を推奨し、
全教職員にAI活用研修を義務化する
```
{{% alert icon="🤖" context="success" %}}
両方の立場から論点を整理しました。

**賛成の立場からの論点：**
- 業務効率化による教職員の負担軽減
- 組織としてのAIリテラシー底上げ
- 先進的な取り組みによる大学のブランディング
- 全学的に統一することで、部署間の格差を防げる

**反対の立場からの論点：**
- 義務化への心理的反発（「やらされ感」）
- 研修時間の確保が業務を圧迫する
- AI活用に懐疑的な教員の理解をどう得るか
- 情報セキュリティリスクへの組織的対応が追いついていない可能性

**見落としがちな観点：**
- **学生への影響**: 教職員がAIを使い始めると、学生のAI利用ルールとの整合性が問われる
- **労働組合・教員組織との調整**: 義務化には事前の合意形成が必要な場合がある
- **推奨と義務化の間**: いきなり義務化ではなく、まず推奨→希望者向け研修→段階的に拡大という選択肢もある
- **効果測定**: 義務化した場合の成果をどう測るか。研修を受けた＝活用できるとは限らない

どの論点を深掘りしたいですか？
{{% /alert %}}

「見落としがちな観点」として指摘された内容は、自分一人では気づきにくいものが含まれているかもしれません。
AIに多角的な視点を出してもらうことで、**検討の抜け漏れを減らせます**。

### - 反論を想定する

```
上の方針案を教授会で提案するとき、予想される反対意見と
それに対する回答案を用意したいです。
厳しめの反対意見を3つ想定してください。
```
{{% alert icon="🤖" context="success" %}}
教授会で予想される厳しめの反対意見と回答案です。

**反対意見1：「教育・研究の自由を侵害するのではないか」**
- 回答案：AIツールの利用を義務化するのではなく、研修の受講を義務化するという趣旨です。研修を受けた上でAIを使わないという判断も尊重されます。知識を持った上での選択と、知らないがゆえの不使用は異なると考えています。

**反対意見2：「そのための時間と予算はどこから出すのか」**
- 回答案：研修は1回2時間程度を想定しており、既存のFD/SD研修の枠組みで実施可能です。外部講師費用については○○の予算から充当する案を準備しています。

**反対意見3：「AIの精度が不十分な段階で推奨するのは時期尚早ではないか」**
- 回答案：AIの精度は確かに完璧ではありません。だからこそ、適切な使い方と限界を理解するための研修が必要だと考えています。使い方を知らないまま個人で利用するよりも、組織として正しい知識を共有する方が安全です。
{{% /alert %}}

## 📝 パターン3：たたき台を改善する
---

自分が作成した文書や計画を、AIにレビューしてもらうパターンです。

### - 研修計画のブラッシュアップ

```
以下の研修計画を作りました。
改善点や足りない要素を指摘してください。
厳しめのフィードバックをお願いします。

---
研修名：AI活用基礎研修
対象：事務職員
時間：2時間
内容：
1. AIの概要説明（30分）
2. ChatGPTの基本操作（30分）
3. プロンプトの書き方（30分）
4. 質疑応答（30分）
---
```
{{% alert icon="🤖" context="success" %}}
研修計画を拝見しました。基本的な骨格はしっかりしていますが、いくつか改善の余地があります。

**構成に関するフィードバック：**
- 前半90分が座学・講義形式に偏っています。参加者が実際にAIを操作する時間がありません。「聞いて分かった」と「使えるようになった」には大きな差があるので、**ハンズオンの時間を設けるべき**です
- 質疑応答30分は長すぎる可能性があります。質問が出なかった場合に間延びします

**足りない要素：**
- **ゴール設定**: この研修を受けた後、参加者は何ができるようになるのかが明示されていません
- **業務との接続**: 「プロンプトの書き方」は一般論ではなく、参加者の実際の業務に即した例を使うべきです
- **リスクの説明**: 情報セキュリティや著作権など、やってはいけないことの説明がありません
- **フォローアップ**: 研修後に参加者が困ったとき、誰に相談すればいいのかが不明です

**改善案：**
1. AIの概要 + 注意点（20分）
2. デモ：実際の業務例で体験（15分）
3. ハンズオン：自分の業務でプロンプトを書いてみる（40分）
4. 共有・振り返り（15分）
5. まとめ + 質疑応答（15分）
6. フォローアップの案内（5分）
{{% /alert %}}

自分で作った計画は、どうしても「これで良い」と思いがちです。
AIにレビューを頼むことで、**第三者の目で見たときの気づき**を得ることができます。

## 🗣️ 壁打ちを上手に進めるコツ
---

### - AIに文脈を伝える

壁打ちの質は、AIにどれだけ文脈を伝えられるかで変わります。

| 伝えると良い情報 | 例 |
|:---|:---|
| 自分の立場・役割 | 「総務課の係長です」 |
| 検討の段階 | 「まだ初期のアイデアです」「ほぼ固まっています」 |
| 制約条件 | 「予算は10万円以内」「3月中に実施したい」 |
| 求めているもの | 「論点を洗い出したい」「別の案も聞きたい」 |

### - AIの回答を踏み台にする

AIの回答は「完成品」ではなく「たたき台」です。
返ってきた内容に対して「ここは違う」「この部分をもっと深掘りしたい」と返していくことで、対話が深まります。

```
さっきの研修計画の改善案、方向性は良いのですが、
ハンズオン40分は初心者には長すぎるかもしれません。
20分の体験 + 20分のペアワークに分けるのはどうでしょう？
```

このように**自分の判断を明確に示しながら対話を重ねる**ことが、壁打ちの本質です。

### - 新しい会話で仕切り直す

壁打ちが長くなると、対話の文脈が複雑になりすぎることがあります。
ある程度まとまったら、**要点を整理して新しい会話で続きを始める**のも有効な方法です。

```
これまでの議論で固まった研修計画を以下にまとめました。
ここから先は、具体的な当日の進行台本を一緒に考えたいです。

【確定事項】
- 研修名：AI活用基礎研修
- 対象：事務職員50名
- 時間：2時間
- （以下、確定した内容を記載）
```

## 壁打ちと対話の技術の違い
---

このページと[対話の文脈を活用する]({{% relref "/docs/intermediate/interactive.md" %}})は、一見似ているようで視点が異なります。

| 観点 | 対話の文脈を活用する | 壁打ち実践 |
|:---|:---|:---|
| 主な目的 | 対話構造を使って回答精度を上げる | 自分の思考を整理・深化させる |
| 主役 | 生成AIの回答の質 | 利用者自身の思考プロセス |
| 典型的な使い方 | 段階的に文書を完成させる | アイデアの検討、論点の洗い出し |
| 成果物 | 精度の高い文書・回答 | より良い判断・意思決定 |

どちらが優れているということではなく、場面に応じて使い分けてください。
文書を仕上げる場面では[対話の技術]({{% relref "/docs/intermediate/interactive.md" %}})を、方針を考える場面では壁打ちを、といった具合です。
