## 📎 ファイル読み込みでできること
---

テキストを手作業で貼り付ける方法と比べて、ファイルを直接読み込ませると以下のようなメリットがあります。

| 方法 | メリット | 制約 |
|:---|:---|:---|
| テキスト貼り付け | 手軽、どのツールでも対応 | 表や図が失われる、長文はコピーが大変 |
| ファイル添付 | レイアウト・表・図を保持したまま分析可能 | ツールごとに対応形式が異なる |

ファイルを読み込ませることで、以下のようなタスクが可能になります。

- **PDF**: 報告書の要約、規程集からの情報抽出、複数文書の比較
- **Excel/CSV**: データの集計・分析・グラフ化、表形式データの変換
- **画像**: 図表の読み取り、手書きメモのテキスト化、写真からの情報抽出
- **Word/PowerPoint**: 文書の校正・要約、スライド内容の整理

## 📄 PDFを読み込ませる実践
---

大学業務ではPDF形式の文書が非常に多く使われます。規程集、答申、報告書、要項など、重要な文書の多くがPDFです。

### - 単一PDFの分析

PDFファイルを添付して、必要な情報を抽出してみましょう。

```
添付したPDFは本学の学則です。以下の観点で整理してください。

1. 休学に関する条項をすべて抜き出す
2. 各条項の要点を箇条書きで要約する
3. 学生が特に注意すべきポイントを3つ挙げる
```
{{% alert icon="🤖" context="success" %}}
添付された学則から、休学に関する条項を整理しました。

**1. 休学に関する条項**

**第28条（休学の要件）**
- 疾病その他やむを得ない事由により3か月以上修学できない場合、休学を願い出ることができる

**第29条（休学期間）**
- 休学期間は1年以内とする
- 特別の事情がある場合は1年を限度として延長を認めることがある

**第30条（休学の上限）**
- 休学期間は通算して2年を超えることができない

**2. 各条項の要約**
- 3か月以上の修学不能が休学の要件
- 1回の休学は最長1年、延長も最長1年
- 合計で2年を超える休学は不可

**3. 学生が特に注意すべきポイント**
1. **申請時期**: 休学は事前申請が原則です。学期途中からの申請は認められない場合があります
2. **通算上限**: 複数回の休学を合計して2年が上限です。計画的な判断が必要です
3. **在学年数への影響**: 休学期間は在学年数に算入されません。卒業時期に影響する点を理解しておきましょう
{{% /alert %}}

### - 複数PDFの比較

複数のファイルを同時に読み込ませると、比較・横断的な分析が可能です。

```
添付した2つのPDFは、A大学とB大学のアドミッション・ポリシーです。
以下の観点で比較表を作成してください。

- 求める学生像
- 選抜方法の特徴
- 重視する能力・資質
- 入学前に求める学習
```
{{% alert icon="🤖" context="success" %}}
2つのアドミッション・ポリシーを比較しました。

| 観点 | A大学 | B大学 |
|:---|:---|:---|
| 求める学生像 | 幅広い教養と専門性を持ち、地域社会に貢献する意欲のある学生 | グローバルな視野を持ち、主体的に学び続ける姿勢のある学生 |
| 選抜方法の特徴 | 学力試験を重視しつつ、面接で意欲を確認 | 多面的・総合的評価を重視、活動実績も考慮 |
| 重視する能力 | 論理的思考力、コミュニケーション力、課題発見力 | 異文化理解力、自己表現力、協働力 |
| 入学前の学習 | 高校の教育課程を幅広く学ぶことを推奨 | 英語力の向上と社会課題への関心を推奨 |

**比較から見える特徴:**
- A大学は地域密着型の人材育成を志向
- B大学は国際性を重視した教育を志向
- 両大学とも主体性・意欲を重視している点は共通
{{% /alert %}}

このように複数文書を同時に扱えることが、ファイル読み込みの大きな利点です。
[基本編]({{% relref "/docs/basic/reading.md" %}})ではテキストを貼り付けて比較する例を紹介しましたが、ファイル添付ならレイアウトや表の情報も保持されるため、より正確な比較が可能です。

## 📊 Excel・CSVを読み込ませる
---

Excel・CSVファイルを読み込ませると、データの集計・分析・可視化をAIに任せることができます。
特にChatGPTの「データ分析」機能やClaudeの「Artifacts」機能では、プログラムを自動生成・実行してグラフを作成することも可能です。

### - データの集計と分析

```
添付したExcelファイルは、過去3年間の学部別入学者数です。
以下を行ってください。

1. 学部ごとの入学者数の推移を表にまとめる
2. 増減の傾向を分析する
3. 定員充足率が90%を下回っている学部があれば指摘する
```
{{% alert icon="🤖" context="success" %}}
添付されたデータを分析しました。

**1. 学部別入学者数の推移**

| 学部 | 2023年度 | 2024年度 | 2025年度 | 定員 |
|:---|---:|---:|---:|---:|
| 文学部 | 198 | 192 | 185 | 200 |
| 経済学部 | 245 | 248 | 252 | 250 |
| 理工学部 | 180 | 175 | 168 | 200 |
| 看護学部 | 82 | 80 | 80 | 80 |

**2. 傾向分析**
- 経済学部は3年連続で微増傾向にあり、安定した志願者を確保しています
- 文学部と理工学部は緩やかな減少傾向です
- 看護学部はほぼ定員通りで推移しています

**3. 定員充足率の確認**
- 理工学部の2025年度充足率は84.0%であり、90%を下回っています
- 文学部の2025年度充足率は92.5%で、基準は満たしていますが注意が必要です
{{% /alert %}}

### - データの変換・整形

```
添付したCSVファイルのデータを以下の形式に変換してください。
- 列の順序を「学籍番号, 氏名, 学部, 学科, 入学年度」に並べ替え
- 入学年度を西暦から和暦に変換
- 結果をCSV形式で出力
```

このようなデータの変換・整形は、手作業では時間がかかりミスも起きやすいタスクです。AIに任せることで効率化できますが、**結果の確認は必ず行いましょう**。

## 🖼️ 画像を読み込ませる
---

多くの生成AIが画像の認識（マルチモーダル対応）に対応しています。
写真、図表、手書きメモ、スクリーンショットなど、様々な画像を分析できます。

### - 図表の読み取り

```
添付した画像は学内アンケートの集計グラフです。
グラフから読み取れる情報を整理し、以下の形式でまとめてください。

- グラフの種類と表示内容
- 主要な数値
- 読み取れる傾向や特徴
```
{{% alert icon="🤖" context="success" %}}
添付された画像を分析しました。

**グラフの種類と表示内容**
- 横棒グラフで、学生満足度調査の5段階評価の分布を表示しています
- 項目は「授業内容」「施設環境」「学生支援」「キャリア支援」の4項目です

**主要な数値**
- 授業内容：「満足」「やや満足」の合計が約72%
- 施設環境：「満足」「やや満足」の合計が約65%
- 学生支援：「満足」「やや満足」の合計が約58%
- キャリア支援：「満足」「やや満足」の合計が約45%

**読み取れる傾向**
- 授業内容への満足度が最も高い
- キャリア支援は他項目と比べて満足度が低く、改善の余地がある
- 全体として「普通」の回答が多く、明確な不満は少ないが積極的な満足も限定的
{{% /alert %}}

### - 手書きメモのテキスト化

```
添付した画像は会議中に取った手書きメモです。
内容を読み取り、構造化されたテキストに変換してください。
読み取りにくい部分は「[不明]」と記載してください。
```

画像の読み取り精度はツールや画像の品質によって異なります。手書き文字、低解像度の画像、複雑なレイアウトの図表では読み取り精度が下がることがあるため、結果の確認が特に重要です。

## 🏢 大学業務での活用パターン
---

ファイル読み込みが特に効果的な大学業務の場面をまとめます。

### - 規程集の横断検索

```
添付した3つのPDF（学則、大学院学則、通則）から、
「成績評価」に関する記述をすべて抽出し、
文書ごとに一覧表にまとめてください。
矛盾や不整合があれば指摘してください。
```

規程の改正作業や自己点検・評価の際に、複数の規程を横断的に確認する作業を効率化できます。

### - 申請書類のチェック

```
添付したPDFは補助金の申請書（記入例）です。
以下の観点でチェックリストを作成してください。

- 必須記入項目が埋まっているか
- 金額の整合性（内訳と合計が一致しているか）
- よくある記入ミスの傾向
```

### - 会議資料の事前整理

```
添付した3つのファイルは次回の教授会の配布資料です。
各資料の要点を5行以内で要約し、
審議が必要な事項と報告事項に分類してください。
```

大量の会議資料を事前に把握しておきたいとき、AIに要約させることで準備の効率が上がります。

## ⚠️ ファイル読み込み時の注意点
---

ファイルの読み込みは便利ですが、以下の点に注意が必要です。

### - 機密情報の取り扱い

{{% alert icon="" context="warning" %}}
<strong>個人情報や機密情報を含むファイルを外部の生成AIサービスにアップロードしないでください。</strong>
学生の成績データ、人事情報、未公表の研究データなどは、学内のセキュリティポリシーに従って取り扱いましょう。
{{% /alert %}}

利用前に確認すべきこと:
- 所属組織のAI利用ガイドラインを確認する
- アップロードしたデータがモデルの学習に使われるかどうかを確認する
- 機密度の高いデータには、組織が契約しているセキュアな環境を利用する

### - ファイルサイズ・ページ数の制限

各ツールにはファイルサイズやページ数の上限があります。大きなファイルを扱う場合は以下を試してみてください。

- ファイルを分割してアップロードする
- 必要なページだけを抽出してから読み込ませる
- テキスト抽出ツールで事前に変換しておく

### - 読み取り精度の限界

- **スキャンPDF**（画像として保存されたPDF）はOCR処理が必要なため、精度が下がることがあります
- **手書き文字**は認識精度にばらつきがあります
- **複雑なレイアウト**（段組み、表と図が混在する文書など）では、情報の読み取り順序が乱れることがあります
- 読み取り結果は必ず**元のファイルと照合**してください

### - 出力結果の検証

ファイルから読み取った情報をもとにAIが生成した内容は、必ず元のファイルと照合してください。
特に数値データや固有名詞は、AIが誤って読み取る可能性があります。

## まとめ
---

ファイルの読み込みは、テキスト貼り付けの延長にある自然なステップです。

- **単一ファイルの分析**から始めて、慣れてきたら**複数ファイルの比較**に挑戦しましょう
- 大学業務では**規程の横断検索**や**会議資料の事前整理**など、日常的な場面で活用できます
- **機密情報の取り扱い**と**読み取り精度の限界**を意識し、結果の検証を怠らないようにしましょう

{{% alert icon="" context="info" %}}
ファイルを読み込ませるスキルは、[基本編の要約]({{% relref "/docs/basic/reading.md" %}})の延長線上にあります。
まだ試したことがない方は、身近なPDF（会議次第や報告書など）から始めてみてください。
{{% /alert %}}
